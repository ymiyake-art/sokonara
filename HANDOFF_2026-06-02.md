## 1. プロダクト概要

**ソコナラ（Sokonara）** = 地域キャリア共創プラットフォームのMVP。
地域企業の経営層の記事をユーザーが読み、価値観アンケート→AI自己分析→経営層との接点（エントリー）を作る、という体験を提供する。

### 主要ファイル構成
- `index.html` … 単一ファイルSPA（ユーザー画面＋簡易管理画面）。3000行超。
- `admin.html` … 本格的な管理画面（企業/記事/興味フォーム管理）。
- `api/feedback.js` … OpenAI（gpt-4o-mini）でAI診断・レコメンド・レポート生成。Edge Function。
- `api/line-token.js` … LINE OAuth トークン交換＋プロフィール取得。Edge Function。
- `api/interest-forms.js` … 管理画面用。service_roleキーで興味フォームをlist/delete/count。Edge Function。
- `images/` … 画像アセット。

### 技術構成
- フロント：HTML/CSS/JS単一ファイル（フレームワークなし）
- バックエンド：Vercel Edge Functions（`export const config = { runtime: 'edge' }`）
- DB：Supabase（PostgREST + Storage + RLS）
- 認証：LINE Login（OAuth 2.0）
- AI：OpenAI gpt-4o-mini

## 2. ユーザー画面の流れ（index.html）

```
p-list（経営層一覧）
  → p1（記事詳細）
    → q1b/q2b/q3b（価値観アンケート3問）
      → p3（AI自己分析フィードバック）
        → p4（レコメンド：似た経営層）
        → 「接点を作る」→ WEBCASフォーム（別タブ）→ p5（完了）
```

### 状態管理
- `const S = { q1, q2, q3, free, fb, screen, mode, adminTab }` … 画面状態（in-memory）
- `selectedCompanyId` … 現在見ている経営層ID
- `goScreen(id)` … 画面遷移
- `SCREENS = ['p-list','p1','p2','p3','p4','p5']`

### アンケート（重要：この設計は確定済み）
- **Q1**：経営層ごとに動的に出る選択肢（記事から1つ選ぶ）→ `S.q1`
- **Q2**：価値観チェック（複数選択）→ `S.q2`
- **Q3**：関わり方（5択・確定済み）→ `S.q3`
  - まず話を聞いてみたい / プロジェクトベースで関わりたい / 将来的な可能性として興味がある / 価値観に共鳴した / まだ整理できていない
  - 直下に「※ 応募ではありません。まずは気軽な対話や情報交換から可能です。」
- **自由記述**：任意。「この経営層に会えたとしたら、最初に何を聞きたいですか？」＋「※記述が多いほどAI診断の精度が上がります」

## 3. AI自己分析（api/feedback.js の `values` ケース）

### 現在の方針（2026-06-02に大改修）
- **断言調**で書く。「〜ではないでしょうか」「〜かもしれません」「〜という印象を受けます」等は**禁止ワードとして明記済み**。
- message は **15行程度**（以前は20行で冗長だった）。
- 価値観3語は「〜な人」「〜できる人」「〜を持つ人」形式。
- ユーザーが入力していない業界知識・専門スキルは**補完禁止**。
- Q1の言葉を必ず引用して読み解く構成。
- JSON返却：`{summary, values[3], message, bridge}`

### 表示ロジック（index.html `showFeedback()`）
- `isLineRegistered()`（= `localStorage.snLineId` の有無）で分岐。
  - **LINE登録済み**：summaryブロック非表示＋detailフル表示（valueTags/message/bridge/CTA）
  - **LINE未登録**：summaryを3文プレビュー＋「続きはLINE登録後に」＋LINE登録CTA
- **注意：かつて `isLineFriend()` を使っていてバグっていた**（友だちチェックは廃止済みなので常にfalse→短い表示になっていた）。`isLineRegistered()` に修正済み。

### フィードバック結果の永続化（2026-06-02実装）
- `genFeedback()` 完了時に `_saveFbCache()` で `localStorage.sn_fb_cache` に保存（companyId/fb/q1-q3/free/ts）。
- `openCompanyArticle(id)` で、同じ企業の24時間以内キャッシュがあれば p3 に直接復元（アンケートをスキップ）。
  - **復元時の順序に注意**：`renderP1FromCompany()` が `S.q1` をリセットするため、render後に `S.q1` を再セットしている。
- フォーム送信完了（p5遷移）時に `localStorage.removeItem('sn_fb_cache')` でクリア。

## 4. LINE Login（重要：ハマりどころ多数）

### 確定事項
- `LINE_REDIRECT_URI = 'https://app.sokonara.co.jp/'`（**固定**。`location.origin+pathname` だとモバイルで値がズレてredirect_uri不一致になるため）
- LINE Client ID：`2009807359`
- 公式アカウント基本ID：`@630zglyb`
- **友だちチェックは完全廃止**。Messaging APIとLogin channelでuserIdのnamespaceが異なり、登録済みでも `friend: false` になってログインが壊れたため。**LINEログイン成功＝アクセス許可** とする。

### フロー
1. `_startLineLoginForContact()`（接点意図あり）or `beginLineLogin()`（FB閲覧）→ LINE認可画面へ
2. stateに `random.c.companyId` 形式で意図を埋め込む
3. コールバックでcode検出 → `/api/line-token` でトークン交換 → `localStorage` に `snLineId/snLineName/snLineFriend` 保存 → `location.reload()`
4. reload後、`sn_postlogin` フラグ＋意図に応じて画面復元

### localStorage キー一覧
- `snLineId` / `snLineName` / `snLineFriend` … LINE情報
- `snLineState` … OAuth state検証用
- `sn_postlogin` … ログイン直後フラグ
- `sn_contact_intent` / `sn_contact_company` … 接点意図
- `sn_restore` … LINE遷移前の状態保存（q1-q3/fb）
- `sn_fb_cache` … AI診断結果キャッシュ（24h）
- `sn_companies` … 企業データキャッシュ

## 5. WEBCAS連携（接点フォーム）— 2026-06-02に大きく変更

### 重要な設計判断
**個人情報はSupabaseに保存しない。WEBCASで収集する。**
プライバシー方針（関西電力等からSupabase/OpenAI/localStorageへの個人情報保存許可が未取得のため）に基づく。

### 現在の動き
- p3/p4の「接点を作る」ボタン → `openEmpathyForm()` を直接呼ぶ（**モーダルは廃止済み**。ワンタップ直行で離脱防止）。
- `openEmpathyForm()` の処理：
  1. Supabaseに**匿名ログのみ**INSERT（line_id, company_id, company_name, ceo_name, intent_type, q1-q3, ai_values, `form_data: {}`空）
  2. `sn_fb_cache` クリア
  3. p5のヒント欄に経営層名をセット（`p5CeoHint` / `p5CeoHintText`）
  4. `window.open(WEBCAS_FORM_URL, '_blank')` で別タブ起動
  5. toast表示 → `goScreen('p5')`
- **WEBCAS_FORM_URL = `https://www17.webcas.net/form/pub/sokonara/pf`**

### p5画面
- 「フォームを開きました」＋WEBCASで詳細入力を促す案内
- **経営層名ヒント**（`p5CeoHint`）：「関心をお持ちの経営層」欄に「石倉 博之さん（石倉建材）」と記入するよう表示
- 「フォームをもう一度開く」「他の経営層の記事を読む」ボタン

### なぜモーダル廃止＆URLパラメータ廃止したか
- モーダル：画面遷移が増えると離脱するため、ワンタップ直行に。
- URLパラメータ（`?company=&ceo=`）：WEBCASのアクセス解析が企業ごとに分散して集計しにくくなるため削除。代わりに**WEBCASフォーム側に「関心をお持ちの経営層」項目を追加**し、ユーザーに手入力してもらう方針。

## 7. Supabase

### テーブル
- `companies` … 経営層/企業。`heroUrl` カラムは2026年に追加（記事ヒーロー写真用、カード写真とは別）。
- `interest_forms` … 興味フォーム。**RLS有効**：anonはINSERTのみ、SELECT/DELETEはservice_role経由（`/api/interest-forms`）のみ。
- `projects` … 共創プロジェクト。
- `reactions` … リアクション。
- `user_sessions` … 旧セッション保存用。**現在 `saveSession()` はno-op（無効化中）**。古いテストデータが残っているが新規書き込みなし。line_idがNULLなのはそのため。
- `users` … LINE登録ユーザー。**空（意図的に未実装）**。LINE情報はlocalStorageのみで管理。

### RLS / セキュリティ方針
- `interest_forms` のSELECTはanon不可 → 個人情報が漏れない。
- service_roleキーはVercel環境変数（`SUPABASE_SERVICE_KEY`）にのみ存在。ブラウザ・GitHubには出ない。
- Storage（`company-photos` / `article-images`）も `storage.objects` にRLSポリシー設定済み（`TO public` で全ユーザーがupload可能に修正済み）。

### 環境変数（Vercel）
`OPENAI_API_KEY` / `LINE_CLIENT_ID` / `LINE_CLIENT_SECRET` / `SUPABASE_URL` / `SUPABASE_SERVICE_KEY` / `ADMIN_PASSWORD`

## 10. プライバシー設計の基本方針（重要・遵守すること）

1. Supabaseには**匿名ID・記事ID・企業ID・回答状態・スコア・AI結果**など個人を直接特定しない情報のみ保存
2. 氏名/メール/電話/勤務先/職歴/自己紹介/自由記述の生データは**Supabaseに保存しない**（WEBCASで扱う）
3. OpenAIには個人特定情報を送らない（送る前にマスク推奨）
4. localStorageに個人情報を残さない（一時保存はsessionStorage優先、送信後削除）
5. 利用規約同意と企業への情報提供・推薦同意のチェックは**分ける**
6. 同意ログ（日時/バージョン/規約URL/識別子）を保存
