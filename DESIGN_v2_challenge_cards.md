# ソコナラ v2 設計書 — 課題カードファースト導線

> ブランチ: `v2-challenge-cards`（Vercelプレビューで検証 / mainは無傷）
> 新URL: `pf.sokonara.co.jp`（v2お披露目用）
> 元提案: ユーザー提示の「アプリ改善提案書」。本書はそれを **MVPに絞り込み**、実装可能な設計に落としたもの。
> 最終更新: 2026-06-09

## 確定した方針（2026-06-09）
- **カードUI**：スワイプ式（2択で不要な方をスワイプ＝捨てる、残す方をタップ/逆スワイプ）。
- **仮想共創メモ**：チップ＋任意の自由記述欄を置く（任意・スキップ可）。
- **カードは共創テーマに紐づく前提**（現状はカードのみ先行。将来は1テーマ=1カード）。
- **LINE登録の位置（折衷案）**：カード・記事・診断のさわりまではLINEなしで見せて温める。**診断のフル詳細を見る所でLINE登録を促す**（ナーチャリング母数を確保しつつ、温まった後に後ろ倒し）。CTA直前まで遅らせない。

---

## 0. なぜ作り直すか（背景）

- LINE登録者に撒いたが反応が薄い。
- 仮説：現行は「記事を読む → いきなり診断 → LINE登録ゲート → 接点」と、**ユーザーの温度より先に"経営層と話す"を要求**している。
- ユーザーは最初から経営層と話したいわけではない。まず **「この課題なら自分の出番があるかも」** という感覚を持たせるのが先。
- 新導線の軸：**課題（出番の想像）→ 人（経営層）→ 接点**。

## 1. MVPスコープ（今回作るもの / 落とすもの）

### 作る（MVP）
1. TOPに新コピー＋「課題カードを選んでみる」CTA
2. 課題カードUI（二択 or スワイプ・3〜5枚）
3. カードのタグから記事レコメンド（**単純タグ一致。推定AIは使わない**）
4. レコメンド記事を読む（既存p1を流用）
5. 記事後の「仮想共創メモ」（チップ選択＋任意自由記述）
6. AIフィードバック（入力に課題カード選択・捨てカード・メモを追加）
7. 面談CTA（**LINE登録はここまで遅らせる**）
8. カード選択・捨て・メモ・到達ログ

### Phase2に落とす（今回やらない）
- 企業入力フォーム（共創テーマ最大3本）§10-12 → 当面は**既存6社から手動でカード抽出**
- 興味テーマ推定AI §7 → タグ一致で代替
- 冒頭ITエンジニア1問 §8 → 「5秒で操作」と矛盾。DX系カード選択で代替
- 別料金オプション §12 → 収益設計。検証に不要
- 「○位：惹かれやすい課題」の表示 → カード結果のサマリー程度に留める

## 2. 画面遷移（v2）

```
v-top（新TOP：コピー＋CTA「課題カードを選んでみる」）
  ↓
v-cards（課題カードを1枚ずつ・二択/スワイプ。3枚で仮、5枚で確定）
  ↓ 選択タグ集計
v-result（あなたが惹かれた課題 上位＋おすすめ経営層記事1本）
  ↓
p1（記事＝既存流用）
  ↓ 読了
v-memo（仮想共創メモ：チップ＋任意記述。スキップ可）
  ↓
p3（AIフィードバック＝既存流用・入力拡張）
  ↓
v-contact（面談CTA。ここで初めてLINE登録 → WEBCAS）
```

旧導線（p-list 記事ファースト）は当面残し、A/B比較できるようにする。

## 3. 課題カードのデータ構造

カードは当面 **コード内 or Supabase `challenge_cards` テーブル**で手動管理。

```
challenge_cards {
  id          text  -- 'card_dx_genba'
  title       text  -- 「紙と電話で回る受発注を、現場に合うDXで変えたい」
  company_id  text  -- 紐づく経営層（記事レコメンド先）
  tags        text[] -- ['dx','engineer'] マッチング兼推定用
  active      bool
  sort        int
}
```

### タグ語彙（7テーマ）
`dx`(DX/エンジニア) / `newbiz`(新規事業) / `local`(地方創生) / `expertise`(専門性) / `orgchange`(組織変革) / `growth`(自己成長) / `global`(海外志向)

### 初期カード（既存6社から手動抽出する叩き台）
- 紙と電話で回る受発注を、現場に合うDXで変えたい → `dx`
- 老舗企業の次の事業の柱をつくりたい → `newbiz`
- 若手が自分から動く組織に変えたい → `orgchange`
- 地域に人と仕事の流れをつくりたい → `local`
- 地方企業の挑戦を海外へ広げたい → `global`
- 職人技を次世代に残したい → `expertise`
- 経営者の右腕として事業を前に進めたい → `growth`

（※各カードを実在6社のどれに紐付けるかは、記事内容を見て確定する）

## 4. レコメンドロジック（MVP・非AI）

1. ユーザーが選んだカードのタグを集計 → 上位タグを算出
2. 上位タグを持つ `company` の記事を提示（1本）
3. 同点・候補複数なら reactions 多い順
4. 「捨てたカード」もログするが、MVPのレコメンドには使わない（ログのみ）

## 5. 仮想共創メモ（v-memo）

見出し：**この会社に関わるとしたら、あなたは何ができそうですか？**
補足：まだ転職を決めていなくても大丈夫です。仮に関わるなら何ができそうか、自由に。

チップ（複数選択）：
業務改善 / IT・DX / 新規事業 / 営業・販路開拓 / 組織づくり / 採用・育成 / 商品企画 / 広報・発信 / 海外展開 / まだ分からない

その下に任意の自由記述欄。**未入力でもAIへ進める。**

## 6. AIフィードバック入力拡張（api/feedback.js）

現行の values 入力に加えて渡す：
- 選んだ課題カード（title配列）
- 捨てた課題カード（title配列）
- 上位タグ（惹かれた課題）
- 読んだ経営層記事（company）
- 仮想共創メモ：チップ＋自由記述
- （Q1〜Q4は新導線では簡略化 or 廃止を検討）

返す内容（§14-15準拠）：
1. ユーザーがどの課題に反応したか
2. 記事内の経営者の考えとの接点
3. この会社でどんな役割を持てそうか
4. 面談で聞くと良さそうな問い
5. 面談CTA

既存の占い対策・主語取り違え厳禁・平易具体・長さ可変は**そのまま継承**。

## 7. LINE登録の位置（重要変更）

- 現行：診断(p3)の手前でLINEゲート（途中カット）。
- **v2：診断はLINEなしでフル表示。LINE登録は面談CTA(v-contact)の直前**に移動。
- 狙い：軽い反応（カード選択・記事・診断閲覧）で20%を取り、重い関門は最後だけ。

```
card_logs {
  id text PK, session_key text,  -- 端末内生成の匿名キー
  shown_cards text[], picked_cards text[], dropped_cards text[],
  top_tags text[],
  memo_chips text[], memo_text text,  -- PIIマスク
  recommended_company text,
  article_read bool, read_pct int,
  ai_viewed bool, contact_cta bool,
  created_at timestamptz default now()
}
```

管理画面で企業ごとに「どのカードが選ばれたか / どの属性が反応したか / どの言葉が面談につながったか」を可視化（Phase2）。

## 9. KPI（§5準拠・軌道修正版）

最重要：**閲覧者の20%が有意味反応**（カード3枚以上 or 記事到達 or AI閲覧 等いずれか）。

目標ファネル：
| 段階 | 目標 |
|------|------|
| 閲覧 | 100% |
| カード開始 | 50% |
| カード3枚 | 35% |
| 記事レコメンド表示 | 30% |
| 記事閲覧 | 25% |
| AI閲覧 | 20% |
| 面談CTA | 5〜10% |

→ 面談CTA単独で20%を狙わず、**AI閲覧/記事到達までを20%へ**。

## 実装状況（2026-06-09 プロトタイプ）
pf/index.html に以下の全画面フローを実装・プレビュー確認済み：
- v-cards（スワイプ・2層カード・共鳴タグ）→ v-result（惹かれた課題ランキング＋レコメンド）
- → **記事**（既存6社の本文をSupabase anonで実取得・画像/ABOUT/WHO付き）
- → **仮想共創メモ**（チップ10種＋任意自由記述・スキップ可）
- → **AI診断**（プロトはJS生成・本番は api/feedback.js へ。スワイプ選択/捨て/メモを根拠に明示）
- → **LINEゲート（折衷案実装）**：診断の「役割＋面談の問い」のフル表示手前で登録CTA。登録で全文＋接点が開く
- → **接点**（WEBCAS呼び出しはプレースホルダ）
※AI診断は現状クライアント生成のスタブ。本番は api/feedback.js に入力拡張が必要（§6）。
