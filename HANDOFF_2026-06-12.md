## 1. 現在のプロダクト構造（v2＝課題カードファーストに大改修済み）

**コンセプト**：求人/経営層探しではなく「**課題との偶然の出会いを通じて、自分の価値観・興味を発見する**」サービス。

### 導線（家1つ・玄関2つ）
```
TOP(index.html / p-list) ＝ 写真ヒーロー＋CTA だけ（顔写真/一覧は省略・サブ化）
 ├「課題カードを選んでみる」(大きいCTA) → /pf/ で即スワイプ開始
 │     pf: カード6枚スワイプ → 理由1問 → 結果(〇〇タイプ＋おすすめ経営層＋偶然の2人目＋記事予告)
 │     → 「記事を読む」→ 既存記事 ?cid=&from=cards&sel=... へ
 └「経営層の記事を一覧で見る」(サブCTA) or ヘッダー無し → 経営層一覧(既定非表示・showLeaderListで表示)

記事(既存p1・要約ファースト：課題/なぜ/求める仲間の3点ダイジェスト→全文)
 → Q1(刺さった言葉・companies.q1から) → 仮想転職(チップ＋自由記述) ※全導線2問に統一
 → AI診断(LINE前=要約, 登録後=要約+本文。登録後も要約を残し連続表示)
 → LINE登録(診断保存＋つながる) → 接点(WEBCASリンク)
```
- ヘッダー(mode-bar): **TOP** / 課題から探す。ロゴ=TOP(goHome)。
- pf の白い説明画面(screenTop)は**廃止**（即スワイプ）。

### ファイル
- `index.html` … メインSPA(TOP/記事/Q1/仮想転職/AI診断/接点/LINE/経営層一覧/admin別)
- `pf/index.html` … 課題カードスワイプ（マッチング入口専用。結果で既存記事へ遷移）
- `admin.html` … 管理画面

- `api/report.js` … service_role集計/削除API。table: user_sessions/empathy_logs/**card_logs**。**action:'get'(パスワード不要・id指定でcard_logs1件=診断復元用)**
