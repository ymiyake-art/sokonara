## 1. 事業ゴール
- **個人（ネオローカルキャリア層）×地域の後継ぎ経営者**をつなぐ。
- 最終CVゴール：**経営者との接点フォーム入力**（カジュアル面談希望等）。
- イベント（ソコナラMEET）はオフライン接点として継続（頻度↓・規模↑）。
- プラットフォーム（アプリ/WEB）をメインの集客・マッチング基盤に。

## 2. 媒体別の役割
| 媒体 | 役割 | CVゴール |
|---|---|---|
| 広告（Meta/Instagram） | 新規流入獲得 | LPへの送客 |
| LP | 興味喚起・ファーストタッチ | アプリ/WEBへの送客 |
| アプリ/WEB | 経営者との接点を作る場 | **フォーム入力（メインCV）** |
| LINE公式 | ナーチャリング・関係継続 | イベント参加・アプリ再訪 |

## 3. 顧客導線
```
【獲得】広告 → LP → アプリ/WEB（フォーム登録＝メインCV）
【育成】          LINE登録 → ステップ配信 → イベント案内・再訪
【深化】          アプリ回帰 → 自己分析（LINE特典）→ 別の経営者と接点
```
### アプリ内フロー（新導線・実装予定）
1. 経営者カードがランダムで複数表示 → ピンときた人をタップ（直感・ゲーム性／「正解選び」にしない）
2. 経営者の記事を読む
3. 「話してみる」フォーム登録 ← **メインCV**
4. 「他にも探す」→ 別カード（回遊）
- **診断/自己分析はメイン導線に置かず、LINE登録後の特典**（CVを診断に食わせない）。

## 4. 差別化コンセプト：「偶然性」（計画的偶発性理論）
- マッチング訴求（競合と同質化）ではなく、**「あなたの転機は、まだ出会っていない人にある」**で差別化。
- LP訴求軸：❌「あなたに合う経営者が見つかる」→ ✅「キャリアの転機は、まだ出会っていない人に」。
- **※ ファーストビューのコピーは「偶然性 vs 具体ベネフィット」をA/Bで検証**（冷たい広告流入では具体型が勝つこともあるため、決め打ちしない）。

## 5. LINE登録への誘導（3段階）※LP上ではLINE誘導しない（LPはアプリ送客に集中）
| タイミング | 文脈 | 温度 |
|---|---|---|
| ① フォーム完了後（サンクス） | 「話した経営者の近況・新着が届く」 | 高 |
| ② 記事は読んだがフォーム未達 | 「合う経営者を探す自己分析はこちら」 | 中 |
| ③ 繰り返し利用後 | 「カード履歴から診断を深める」 | 中 |

## 6.【計測・分析】現状と役割分担 ★マーケ会社と特に握りたい部分
### 計測は“未確定”ではなく**構築済み**
アプリ側に独自計測が入っています（本番で動作確認済み）。GA（マーケ側）と二重で持ち、**乖離チェック**に使います。

**(a) 自社・行動ファネル（管理画面「共鳴データ」／card_logs）**
カード開始 → 3枚以上スワイプ → 診断結果に到達 → 記事を読んだ → AI診断を見た → LINE登録 → 面談CTA を、**チャネル別（utm_source）**で集計。経営層別・惹かれたテーマ・共鳴タイプも。個人情報なし。

**(b) Vercel Web Analytics**：訪問数・PV・流入元(UTM/参照元)・デバイス・地域（Cookie/PIIなし）。

**(c) GA4（マーケ会社運用）**：広告→LP→アプリの送客、LP直帰、コンバージョン経路など。

### UTM命名規約（広告・各リンクで統一）
| パラメータ | 意味 | 例 |
|---|---|---|
| `utm_source` | 流入元 | `meta_ads` / `instagram` / `line` / `email` / `bank_article` / `personal_lp` |
| `utm_medium` | 種別 | `cpc` / `social` / `message` / `email` / `article` / `lp` |
| `utm_campaign` | 施策・時期 | `202607_meta_launch` など |
| `utm_content` | 位置・バリエーション（A/B） | `hero_a` / `hero_b` / `app_cta` |
- 入口は原則 **`https://app.sokonara.co.jp/?utm_...`**（`/`で受けて`/pf`へUTMを引き継ぐ実装済み）。LP経由は `/personal/?utm_...` も可。
- **社内テストは `utm_source=admin`** でタグ付け → 双方で除外して比較。

### 役割分担と「数値の乖離チェック」
- **マーケ会社（GA）**：広告〜LP〜アプリ送客の最適化（基本お任せ）。
- **自社（card_logs／Vercel）**：アプリ内の深いファネル（記事→AI診断→フォーム）とチャネル別の“深さ”。
- **我々のチェック観点（乖離が出たら原因を疑う）**：
  - 訪問数：GA vs Vercel（タグ貼り漏れ／二重計測／ボット差）
  - チャネル別流入：GAの参照元 vs 自社の`utm_source`件数（UTM未付与リンクが無いか）
  - フォームCV：GAの目標 vs 自社の面談CTA/フォーム到達（イベント定義のズレ）
  - 社内テスト（`utm_source=admin`）が双方で除外できているか
