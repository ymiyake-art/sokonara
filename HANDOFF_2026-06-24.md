## 1. 計測（★今セッションで大改修・最重要）
- **card_logsへの書き込みは全て service_role API `/api/card-log`（upsert）経由**。
  - 理由：**anonのUPDATEがRLSで「204だが0行更新」**になる不具合があり、マイルストーン(スワイプ/到達/記事/AI/LINE/面談)も診断保存も全滅していた。APIでバイパスし**本番実証済み**。
  - 呼び出し元：pf=`_saveLog/logStart/logUpdate`、index=`patchCardLog/saveDiagForRestore`。
  - → 旧SQL `card_logs_fix_update_policy.sql` は**不要**（APIで解決済み。RLSに依存しない）。
- ファネル（admin「📊 共鳴データ」）：カード開始→3枚以上スワイプ→診断到達→記事→AI診断→LINE登録→面談CTA。**チャネル別(utm_source)**も。`started=false`の復元専用行は集計から除外。
- **Vercel Web Analytics** 導入済(index/pf/biz)。ダッシュボードで有効化済。
- **UTM**：utm_source/medium/campaign/content を card_logs に記録。入口`/`でも `/pf` へ引き継ぐ（localStorage `sn_utm` ＋ goCards付与）。**社内テストは `utm_source=admin`** でタグ付け→集計から除外して比較。

## 2. AI診断（`api/feedback.js`）
- 2タイプ：`cardfeedback`(課題カード経由)／`values`(直接導線)。両方 **gpt-4o＋ストリーミング中継**（Edge関数の504回避）。index.htmlは**逐次表示(タイピング)**（`partialJsonStr`）。
- ルール：JSON厳格化／長さ目安12〜18行／おすすめ経営層と別社カードの取り違え防止／**自由記述・本人の言葉を必ず反映**（一般論禁止）。
- 記事ルールの**正本はNotion**（リッチ）。app の cardgen/articlemeta は「Notion準拠の下書き器」。
- **LINE登録前後で診断一致**：生成時にcard_logsへ ai_summary/message/bridge を保存し、OAuth `state` に logId(または vsId) を載せ、復帰時 `/api/report` action=get で**同一診断を復元（再生成しない）**。直接導線(values)も `started=false` 行で対応。
- 自由記述欄に「スマホ音声入力が楽」ヒント追加済。
- CTA文言：LINEは「保存」→「続きを読む／経営層とつながる」に修正済。
